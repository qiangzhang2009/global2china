## 三、核心组件选型（云端优先·成本最低·可无限复用）

### 1. 多网站接入层：统一前端埋点/工具交互/表单提交SDK

- **开源免费SDK**：

    - 埋点SDK：**PostHog JS Lite（Apache 2.0，永久免费）**，轻量级，支持多租户，支持自定义事件，完全覆盖你的用户行为/工具交互/表单提交数据采集需求

    - 工具交互SDK：基于PostHog JS Lite二次开发（仅需100行代码），支持全量采集工具输入参数、结果页行为

    - 表单提交SDK：基于PostHog JS Lite二次开发（仅需50行代码），支持自动采集表单数据，支持脱敏

- **部署方式**：直接嵌入到各个网站的HTML头部，无需额外部署

- **成本**：完全免费

### 2. 多租户数据隔离层：Cloudflare + Supabase + Upstash（免费额度最高的云端组合）

## 四、可无限复用的核心逻辑（新建网站只需3步）

### 1. 第1步：嵌入统一SDK（10分钟）

- 直接将统一前端埋点/工具交互/表单提交SDK嵌入到新网站的HTML头部

- 在SDK中配置新网站的「租户ID」和「业务规则」（如需要采集的自定义事件、需要脱敏的字段）

### 2. 第2步：分配租户权限（5分钟）

- 在统一后台管理系统的「租户管理后台」中创建新租户

- 为新租户分配「业务配置权限」「数据管理权限」「用户管理权限」

- 为新租户的管理员创建账号

### 3. 新租户开通 Checklist
创建新租户时，显示必填项清单，追踪码配置是**第一优先级**：
- [ ] 嵌入统一SDK
- [ ] 配置追踪码
- [ ] 验证数据采集正常

### 4. 追踪码强制校验机制（核心！）

在**租户管理后台**中实现强制追踪码验证：

#### 4.1 数据库层面
- 租户表增加字段：
  - `tracking_code_status`: 追踪码状态（pending/active/inactive/error）
  - `tracking_code_verified_at`: 最后验证时间
  - `tracking_code_first_verified_at`: 首次验证成功时间

#### 4.2 租户创建流程强制校验
1. **添加租户时**：必须填写追踪码嵌入代码
2. **自动验证**：系统自动检测追踪码是否生效（发送测试事件，验证是否收到数据）
3. **验证通过前**：租户状态为 "待验证"，无法使用完整功能
4. **定期复检**：每天自动验证所有租户的追踪码状态

#### 4.3 告警机制
- 未添加追踪码的租户：显示醒目的警告标识
- 追踪码失效：立即通知管理员
- 数据采集异常：自动告警

#### 4.4 追踪码验证技术实现

// 验证逻辑伪代码
async function verifyTrackingCode(tenantSlug) {
  // 1. 生成唯一的测试事件ID
  const testEventId = `verify_${Date.now()}_${Math.random()}`

  // 2. 通过追踪API发送测试事件
  await fetch('/api/tracking', {
    method: 'POST',
    body: JSON.stringify({
      event_type: 'verification_test',
      tenant_slug: tenantSlug,
      verification_id: testEventId
    })
  })

  // 3. 等待5秒让数据写入
  await sleep(5000)

  // 4. 查询是否收到测试事件
  const result = await sql`
    SELECT COUNT(*) FROM tracking_events
    WHERE tenant_id = ${tenantId}
    AND event_data->>'verification_id' = ${testEventId}
  `

  return result > 0
}

## 七、避坑提示

1. **Supabase PostgreSQL的免费额度读请求是0.00002/次，不是0.0002/次**，我之前写错了，Pro版更划算

2. **Upstash Kafka Lite的BSL协议**，内部后台使用完全免费，商用对外提供服务需在3年后开源对应代码（仅Kafka Lite自身代码，你的业务代码无需开源）

3. **多租户数据隔离**：一定要在Supabase PostgreSQL中用「Row Level Security（RLS）」实现严格的多租户数据隔离，避免不同网站的数据泄露

4. **数据备份**：Supabase PostgreSQL每天自动做全量备份，保留7天，Cloudflare R2支持版本控制，Upstash Redis支持自动备份，一定要定期做恢复测试

5. **性能优化**：Supabase PostgreSQL中要建合适的索引，Upstash Redis中要合理设置缓存时间，避免缓存雪崩

### 8. 数据脱敏规范（补充）

```typescript
// 数据脱敏规则示例
const desensitizeRules = {
  // 手机号：只显示前3位和后4位
  phone: (value: string) => value.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2'),
  
  // 邮箱：只显示@前的第一位和最后一位
  email: (value: string) => value.replace(/(.).*(@)/, '$1***$2'),
  
  // 姓名：只显示姓
  name: (value: string) => value.charAt(0) + '**',
  
  // 聊天内容：敏感词过滤
  message: (value: string) => filterSensitiveWords(value),
  
  // 企业名称：脱敏处理
  company: (value: string) => value.charAt(0) + '***' + value.charAt(value.length - 1)
};
```
